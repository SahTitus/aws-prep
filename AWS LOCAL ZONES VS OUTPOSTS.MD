# AWS Local Zones vs AWS Outposts

## AWS Local Zones
**AWS-managed infrastructure extensions** in **metro areas** that bring **select AWS services** closer to **end users** for **single-digit millisecond latency** applications.

### Key Benefits / Purpose
- **Low latency** for end users  
- **Real-time applications** (gaming, media streaming, live video)  
- **Extension of AWS Region**  
- **No infrastructure management required**  

## AWS Outposts
**AWS-managed infrastructure** deployed at your **on-premises facility** to run **AWS services locally**, ideal for **data residency requirements** or workloads that must remain **on-site**.

### Key Benefits / Purpose
- **Data residency and compliance requirements**  
- **Local data processing**  
- **Hybrid cloud deployment**  
- **Low latency** for on-premises applications  
- **Consistent AWS experience on-premises**
